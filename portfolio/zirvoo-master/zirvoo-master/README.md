# Zorvyn Finance System Backend\n\nA FastAPI-based backend for finance tracking, implementing assignment requirements for transactions CRUD, summaries, role-based access control, validation, and SQLite persistence.\n\n## Project Structure\n\n- `app/`\n  - `main.py` - FastAPI application with routes and RBAC dependency checks\n  - `models.py` - SQLAlchemy models (`User`, `Transaction`)\n  - `schemas.py` - Pydantic schemas for request/response validation\n  - `crud.py` - Business logic for CRUD operations and summaries\n  - `database.py` - SQLite engine and session management\n  - `dependencies.py` - Role-based guard dependencies\n- `requirements.txt`\n- `README.md`\n\n## Setup (local)\n\n1. Create and activate Python virtual environment (Windows):\n   ```powershell\n   python -m venv venv\n   .\\venv\\Scripts\\Activate\n   pip install -r requirements.txt\n   ```\n\n2. Start server:\n   ```powershell\n   uvicorn app.main:app --reload --host 0.0.0.0 --port 8000\n   ```\n\n3. Open API docs:\n   - Swagger: `http://127.0.0.1:8000/docs`\n   - Redoc: `http://127.0.0.1:8000/redoc`\n\n## Role-based Access Control\n\n- Use header `X-User-Role` with one of: `admin`, `analyst`, `viewer`\n- `admin`: full access (users + transactions + summary)\n- `analyst`: can create/update/view transactions + summary\n- `viewer`: can view transactions + summary only\n\n## API highlights\n\n### Users\n- `POST /users` (admin only)\n- `GET /users` (admin only)\n\n### Transactions\n- `POST /transactions` (admin, analyst)\n- `GET /transactions` (viewer, analyst, admin) filters: `type`, `category`, `start_date`, `end_date`, `owner_id`\n- `GET /transactions/{id}`\n- `PUT /transactions/{id}` (admin, analyst)\n- `DELETE /transactions/{id}` (admin only)\n\n### Summary\n- `GET /summary` (viewer, analyst, admin)\n  - total income, total expense, balance\n  - category wise profit/loss\n  - monthly totals\n  - last 10 records\n\n## Example requests\n\n- Create transaction payload:\n  ```json\n  {\n    "amount": 1200.5,\n    "type": "income",\n    "category": "salary",\n    "date": "2026-04-01",\n    "notes": "April salary",\n    "owner_id": 1\n  }\n  ```\n\n- Get filtered expenses by category:\n  `/transactions?type=expense&category=food`\n\n## Notes\n\n- On first run, seed users are created: `admin`, `analyst`, `viewer`.\n- SQLite file created at `./app.db`.\n- This is a minimal single-module backend designed to look hand-crafted and maintainable.\n
